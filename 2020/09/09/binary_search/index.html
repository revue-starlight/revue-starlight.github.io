二分答案/二分专题

模板

origin : 模板源自去年上课，但是tourist也是这么写的。所以非常好。

右边是可行区间

1. l,r边界
保证r时一定可以取到。否则会返回r，尽管r没有用
mid=(l+r)/2;
if (ok) r=mid; else l=mid+1
若不能保证r一定可以取到，则取r+1为r边界。这样如果返回r+1，则是全区间无解。
l必为不可行或最优可行，不然会存在比l小的更优解被忽视。也就是没有比l小的更优解即可

2. mid
mid指针指向不可行区间一侧，mid不可行就踢过来，否则r=mid;

右边是可行区间

同上

形如lower_bound的二分

lowerbound返回arr[i]中第一个大于/等于x的指针。
不难看出是找大于等于的最左端，也就是可行区间在右侧。
check()函数判断这个值是否大于等于x。
⭐有可能全区间无解。此时可行区间侧指针+1。

来源：[kuangbin带你飞]

A.codeforces 371C

某汉堡要原料：Bread Sausage Cheese 各有一些材料，三种材料各有价格，有一些钱，商店货物无限。给一字符串–做一个汉堡的原材料需求。 求最多能做几个汉堡

  1. 对汉堡数进行二分。此处0个汉堡一定可行，可行区间在左边，不可行区间为钱数+已有汉堡数后。
  2. l=0，r=100+money;

l=1 WA 9
r取了1e12(money MAX)，少加了原材料可做的100 WA 22

B.codeforces 343C

一个线性的磁盘有几个位于arr[i]位置的磁头接头，有几个arr2[i]的待扫描点。对于每个都可以在每秒移动1个位置，且每个都是同时运动的，且不用考虑重叠问题，问：最少多少秒可以每个都扫完。
贪心：每个人干完事情后再干点事情。
考虑每个点。选左边吧（右边有点麻烦）每个人的必须任务是完成最左边的点，然后各尽所能的压到右边的那个点去。
一个就是有两种方法，在往左到index的情况先往右再往左，要么就是先往左再往右。选取最优状态。
这里可以参考tourist的代码。有必完成的a，起始位置s，对结束位置b二分。若 s < a 则a=s。
a s b

结束位置b在s和n之间二分，采用左区间可行。mid=(l+r+1)/2.若返回左节点且左节点不合法 return 0;
若返回n return 1;
但是tourist把b的二分区间设定为[index,n];。（大概是8分钟狂写不考虑小复杂度了吧）
这层出来直接把index设为l。
然后最后出来如果index==n 即可行，外层r=mid，否则 l=mid+1;

C.POJ 3685 Matrix

神仙题，给一公式和排名，求值。不难发现对于每一列从上向下单调，所以二分答案，再在队列中二分答案有多少给数比这个数小—也就是左边可行区间里最大的那个数字。也可以是右边的最小。要注意的是这个数字有可能比所有数字都大。
左边可行的最大数字

事后看就是一个lower_bound改，感觉挺难的，调了好久。
再对外面进行二分答案。
统计一共比它小的数字tot, (tot >= m-1 ： TRUE|| tot < m-1 ： FALSE) 最后l,r要收束在tot-1>=m的第一个位置上。 可行区间是右半边的第一个。然后l=min(calc()) r=max(calc()) – 这样没问题因为若第一名一定可行因为一直返回true，r直接移到左。r=l也返回true。最后一名直到r-1也是false，所以变成r且必为true。

D.POJ 3104 Drying

有一炉，一日，濡湿衣物若干。置衣日中，每疏忽减1水，置衣炉上，每疏忽减k水。炉但可置一衣于上。
最少多久可以全部晒干呢？
二分答案，最少0秒，最多max(arr)秒，二分一下，可行区间当然是右边了。
贪心区间：
入门好想。每次全部先减下k秒，然后对>0的剩余物进行m+(k-2)/k-1 ⭐小心爆0⭐ （+k-2的原理 令k=4 则(1，2，3，4)/4都等于1。以此类推。）

⭐scanf()效率远低于 cin+关闭输入输出流⭐ wa了一小时

E.POJ 3579 Median

给出n个数字，找出所有∣ai−aj∣| a_i-a_j |∣ai​−aj​∣,放到集合X中，然后问X中排名为x的这个数字是多少。
很毒瘤，排名为x，也就是有 < x个数字比之小 l=mid+1，但是倘若有 >= x个数字比之小，那么r=mid,最后收束在某个点，这个是第一个>=x个数字比之小呢还是最后一个< x个数字比之小呢?
如果是第一个的话.

找< x-1 个数字比之小，>=x-1 的第一个数字呢？
1 2 2 2 3 中间的数字为2，x=3，(1,0),(2,1),(3,4)。但是不是0，因为本来应该是都为排名2的这个数字，堆砌过了2。所以往后挤了。
所以应该找最后一个 “有< x个数字比之小” 的数字。
就可以二分了。
